HTML

  • The head element contains metadata that helps the browswer render the page correctly.
  • The body element represents the visible content shown to the user.
  • An HTML attribute defines additional information about the element.
  • Metadata contains information for the browser about accessibility, search engines and performance.
  • The Title element defines the name of the page.
  • The style element defines the CSS style.
  • The base element defines the base URL for the page.
  • The link element connects an external resource to the HTML document.
  • The script tag embeds client-side scripts into the HTML document.
  • Elements nested inside other elements are refered to as child elements.
  • Elements with other elements nested inside them are refered to as parent elements.
  • Semantic HTML elements are those whose name describes the element.
  • The Footer is located at the bottom of the page and often contains info on the author, copyright, contact information, sitemap and navigation.

CSS

  • CSS stands for Cascading Style Sheets.
  • There are 3 ways to use CSS to stylize a website: inline CSS, internal (or embedded) CSS and external CSS.
  • The external CSS file is the most efficient method, since many pages can be styled by the same CSS file.
  • A selector defines the element to which the declaration will apply.
  • Declarations should be separated by semi-colons. Multiple can be used after one selector.
  • Declarations contain 2 components: The CSS property we want to apply and the value of the property.
  • A class selector is a special wildcard symbol used to apply to a group of elements.
  • A class selector is assigned in HTML.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A special wildcard symbol such as * can be used that will apply the rule to all elements on the page.

Git

  • git command must preceed every git statement
  • git status: checks what branch we are currently on
  • git checkout *insert branch name*: change the branch from one to another
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add: adds modifications in the current working branch to the staging area
  • git add -A: adds all changes
  • git pull: receives branch modifications into the local environment
  • Always do git pull before modifying code to ensure your local environment is up to date
  • git commit: commit files
  • -m: used after git commit to include a message
  • git push: uploads local changes to GitHub

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • var keyword declares a variable.
  • After declaring a variable, give it a value.
  • = is the assignment operator that assigns specific data we want stored in our variable.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript elements should be at the end of the HTML document because the HTML needs to be present first.
  • Each line of JavaScript ends with a semi-colon.
  • JavaScript is case sensitive.
  • Variables can hold different data types: String, Number, Boolean, Array or Object.
  • A conditional statement or loop can help a developer deviate a program from the linear flow of an application.
  • Console.log can be used to test JavaScript code.
